# Math (ป.๑) lesson-plan variant

PDF แผนคณิตศาสตร์ ป.๑ มีโครงคนละแบบกับภาษาไทย จึงมี pipeline แยกใน `scripts/`:
`extract_math_pdf.mjs` → `build_math_docx.mjs` → `validate_math_docx.mjs` (รับ `<planDir>` + อ่าน `config.json`).

## โครง PDF ที่เรนเดอร์ (faithful)
- **header**: บรรทัด ๑ "แผนการจัดการเรียนรู้" / ๒ "กลุ่มสาระ… ชั้น…" / ๓ "หน่วยการเรียนรู้ที่ N … เวลาเรียน H ชั่วโมง"
  + extra: "แผนการจัดการเรียนรู้ที่ N เรื่อง … เวลาเรียน 1 ชั่วโมง", "สอนวันที่…" (แยกหัวข้อ/เวลาเป็นสองคอลัมน์)
- **pre-table** (เลขอารบิก): มาตรฐานการเรียนรู้/ตัวชี้วัด • สมรรถนะหลัก • จุดประสงค์การเรียนรู้สู่ตัวชี้วัด (K/P/A) •
  สาระสำคัญ • สาระการเรียนรู้ • คุณลักษณะอันพึงประสงค์ • ชิ้นงานหรือภาระงาน
- **ตารางกิจกรรม ๓ คอลัมน์**: กิจกรรมการเรียนรู้ | เวลา (นาที) | สมรรถนะ
- **media**: หัวข้อ "สื่อการเรียนรู้" (เก็บ heading จริง เผื่อสะกดต่าง)
- **assess (list)**: การวัดและประเมินผลการเรียนรู้ — ๑ วิธีการ / ๒ เครื่องมือ / ๓ เกณฑ์
- **tail (template คงที่ทุกแผน — ยืนยันด้วย hash)**: ตารางแบบสังเกตพฤติกรรม ๒ ใบ (๒๐ แถว) + ลงชื่อ ผู้ประเมิน×๒,
  บันทึกหลังการจัดการเรียนรู้ (ผลที่เกิด/ปัญหา/ข้อเสนอแนะ) + ลงชื่อ ผู้สอน + ผู้บริหารสถานศึกษา

## builder: กฎ layout ที่ต้องคงไว้ (แก้ตามฟีดแบ็กผู้ใช้แล้ว — อย่าถอยกลับ)
- **pre-table keepNext**: ผูก `keepNext` เฉพาะ "หัวข้อ (heading) → บรรทัดแรกใต้มัน" เท่านั้น
  **ห้ามผูกทุกบรรทัด** — ถ้าผูกหมด Word มองทั้งบล็อกเป็นก้อนเดียวที่แตกหน้าไม่ได้ แล้วดันไปขึ้นหน้าใหม่
  ทำให้**หน้าแรกเหลือแค่หัวเรื่อง** (อาการที่ผู้ใช้รายงาน)
- **ตารางกิจกรรม = FIXED layout + columnWidths ตายตัว** (twips): `ACT_COLW=[5350,1300,2422]` รวม 9072
  (= ความกว้างใช้งาน A4 portrait, margin 2.5cm). เซลล์ใช้ `WidthType.DXA` + `TableLayoutType.FIXED`.
  ถ้าใช้ % หรือ auto → คอลัมน์กิจกรรมกินพื้นที่หมด คอลัมน์เวลา/สมรรถนะถูกบีบจนตัวอักษรตกแนวตั้ง (อาการที่ผู้ใช้รายงาน)
- **เส้นจุดบันทึกหลังสอน (`....`) = เต็มบรรทัด**: ใช้ `dottedFull()` = paragraph ที่มี
  right tab ที่ `TabStopPosition.MAX` + `LeaderType.DOT` (ขีด `\t` หนึ่งตัว) → จุดยืดเต็มความกว้างเหมือน PDF.
  **ห้ามใช้จำนวนจุดคงที่** (`".".repeat(n)`) เพราะไม่เต็มบรรทัดและความยาวเพี้ยนตามฟอนต์.
- ลายเซ็น: "ลงชื่อ ....(38).... <role>" ใช้จุดคงที่ได้ (เป็นช่วงสั้นกลางบรรทัด ตรงกับ PDF)
- pageBreakBefore: หัวตารางสังเกต ๒ ใบ + บันทึกหลังสอน (รวม 3)
- **ตัดคำไทยไม่ขาดกลางคำ (สำคัญในเซลล์ตารางแคบ) — ต้องทำ ๒ อย่างคู่กัน:**
  1. ตั้ง `language: { value: "en-US", bidirectional: "th-TH" }` ในทุก run + default style (`<w:lang w:bidi="th-TH"/>`)
     — ช่วยเรื่อง shaping/spacing แต่ **ลำพังไม่พอ**.
  2. **แทรก ZWSP (U+200B) ที่ขอบเขตคำไทยทุกคำ** ผ่าน `thaiWrap()` ใน helper `run()` —
     ใช้ `Intl.Segmenter("th",{granularity:"word"})` (มีใน Node, พจนานุกรมไทย) แตกคำแล้วคั่นด้วย ZWSP.
     เหตุผล: ไทยไม่มีช่องว่างระหว่างคำ Word เลยมองทั้งประโยคเป็น token เดียว **บังคับตัดกลางคำ**
     ในเซลล์แคบ (เช่น "สรุปผลการประเ/มิน", "ความสา/มารถ", "เล/ขที่"). ZWSP บอก Word ว่าตัดได้เฉพาะระหว่างคำ.
  - `thaiWrap()` ทำเฉพาะ string ที่มีตัวอักษรไทย และคั่น ZWSP เฉพาะหน้า segment ที่เป็นไทย (ไม่แตะเลข/วงเล็บ/ช่องว่าง).
     ZWSP ไม่อยู่ในช่วง PUA → ไม่กระทบตรวจ ๐ PUA. **validator ต้อง strip ZWSP** ก่อนเทียบ `.includes()` ชื่อ/หัวข้อ.

## extractor: จุดที่เคยพลาด (อย่าทำซ้ำ)
- **PUA/normalizeThai** ต้อง reuse (สระ/วรรณยุกต์ TH SarabunPSK อยู่ใน PUA)
- **เลขหน้า**: ตัดเฉพาะบรรทัดที่เป็น digit ลำพังตัวเดียว x>480 (อย่าตัด digit ที่อยู่บรรทัดเดียวกับข้อความ เช่นเลขชั้น "1" x≈512)
- **คอลัมน์ตารางกิจกรรม**: act x<300, time = digit ที่ x∈[312,365), comp x≥365
- **แมปแถวกิจกรรม**: y-band (anchor=บรรทัด c1 ที่ขึ้นต้น `\d+\.\S`, EPS=8) เพราะเลขเวลา+สมรรถนะบรรทัดแรกของ step ถัดไป
  ลอยสูงกว่าเลขข้อ ~1px (merge ตรงๆ จะหลุดไป step ก่อนหน้า → time ปนข้อความไทย/แถวเพี้ยน)
- **header ตารางซ้ำทุกหน้า**: ตัดบรรทัด include "เวลา (นาที)" หรือ startsWith "กิจกรรมการเรียนรู้" หรือ === "สมรรถนะ"
- **จบ activity**: section ถัดไปอันแรกสุดใน [media, assess, obs] (อย่า hardcode ว่า media มาก่อนเสมอ — ถ้า idxMedia=-1
  ตารางจะลามไปจับเลขข้อหัวข้ออื่น เกิดแถวผีว่าง)
- **config มี BOM**: PowerShell `Out-File -Encoding utf8` เขียน BOM → `loadConfig` ต้อง strip `﻿` ก่อน JSON.parse

## config.json ต่อแผน
`subject, grade, numerals:"arabic", font, fontSizePt, pdfName, outputName, evaluator, instructor, directorLabel, school, worksheetCount:0`

## batch (loop ใน PowerShell — ไม่มี run_all script แล้ว)
ต่อ PDF ใน `10แผนคัดสรร\`: สร้างโฟลเดอร์ `<base>\` ใต้ `แผนคณิตศาสตร์\` → copy pdf → เขียน config →
`node extract_math_pdf.mjs <planDir>` → `node build_math_docx.mjs <planDir>` → `node validate_math_docx.mjs <planDir>`.

## ข้อควรรู้สภาพแวดล้อม
- bash ใน env นี้ **รีเซ็ต cwd หลังทุกคำสั่ง** และ resolve path สคริปต์ไทยพลาด → ใช้ **PowerShell** + absolute path
- **อย่ายิงหลายคำสั่ง parallel** — ถ้าตัวหนึ่ง error ทั้งชุดถูกยกเลิก
- เครื่องไม่มี Word/pdftoppm → ตรวจได้ระดับ XML เท่านั้น; ผู้ใช้เปิด Word ยืนยันหน้า/ตารางจริงเอง
