# กฎจัดหน้า keep-together (ห้ามหัวข้อ/ตารางขาดหน้า)

ปัญหาที่แก้: เวลาขึ้นหน้าใหม่ หัวข้อค้างท้ายหน้าเดิมแต่เนื้อหา/ตารางไปต่อหน้าถัดไป (ตารางถูกตัดครึ่ง /
หัวข้อโดดเดี่ยว) หลักการ: **ถ้าบล็อกจะถูกตัด ให้เลื่อนหัวข้อลงไปอยู่หน้าเดียวกับเนื้อหา** (พฤติกรรม `keepNext` ของ Word)

กฎเหล่านี้ฝังอยู่ใน `build_docx.mjs` แล้ว (เป็น DEFAULT ทุกแผน/ทุกวิชา) — สรุปไว้เพื่อเข้าใจ/ตรวจสอบ:

| องค์ประกอบ | property | ผล |
|-----------|----------|-----|
| ทุก section (๑–๘) | `keepNext: true` ทุกบรรทัด **ยกเว้นบรรทัดสุดท้ายของ section** + `keepLines: true` | หัวข้อ + ทุกข้อย่อย = บล็อกเดียว ถ้าท้ายหน้าไม่พอ Word ดันทั้ง section ลงหน้าใหม่ (แต่ละ section < ๑ หน้า จึงพอดีเสมอ) บรรทัดสุดท้ายไม่ผูก → ยอมขึ้นหน้าระหว่าง section |
| ตาราง ๙ การวัดผล | ทุก `TableRow.cantSplit: true` + ทุกย่อหน้าในเซลล์ `keepNext: true` (helper `tp()`) | หัวข้อ ๙ + ทั้งตารางผูกเป็นบล็อกเดียว ไม่ถูกตัดข้ามหน้า |
| บันทึกหลังแผนการจัดการเรียนรู้ | หัวข้อ `pageBreakBefore: true` | ขึ้นหน้าใหม่เสมอ รวมข้อ ๑/๒/๓ + ลายเซ็นผู้บันทึกไว้ด้วยกัน |
| ตารางลายเซ็น ๓ บล็อก | `TableRow.cantSplit: true` | บล็อกลายเซ็นไม่ขาดหน้า |

**กฎเหล็ก:** บรรทัด**สุดท้าย**ของแต่ละ section ต้อง `keepNext: false` เสมอ — ไม่งั้นจะผูกติดหัวข้อ section ถัดไปเป็นโซ่ยาวทั้งแผน

## ข้อจำกัด / การตรวจสอบ
- `keepNext` / `cantSplit` / `pageBreakBefore` เป็น OOXML มาตรฐานที่ **Word ทำตาม** แต่ตัวสร้าง docx ไม่ render หน้าจริง
- เครื่อง dev ส่วนใหญ่**ไม่มี Word/LibreOffice** → ตรวจได้แค่ว่า property ถูกเขียนลง `word/document.xml`
  (`validate_docx.mjs` นับให้: `cantSplit` = ๗ × จำนวนแผน [๔ แถวตาราง ๙ + ๓ แถวลายเซ็น], `pageBreakBefore` = ๑ × จำนวนแผน)
- **ผลการแบ่งหน้าจริงต้องให้ผู้ใช้เปิดใน Word ตรวจ** — แจ้งผู้ใช้ทุกครั้ง
- เงื่อนไขที่กฎนี้ใช้ได้: ทุก section สั้นกว่า ๑ หน้า ถ้า section ใดยาวเกิน ๑ หน้า Word จะ fallback ตัดให้เอง (ไม่ค้าง loop)
