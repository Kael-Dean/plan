# Layout / รูปแบบเอกสาร — หน่วยการเรียนรู้ที่ ๔ เรื่อง กลัวทำไม

> ไฟล์นี้บันทึกรูปแบบของไฟล์ PDF ต้นฉบับ `หน่วยการเรียนรู้ที่ ๔ กลัวทำไม.pdf`
> และขั้นตอนแปลงเป็น DOCX (รูปแบบเดียวกับบทที่ ๑ น้ำใส / ๒ ใจหาย / ๓ ครัวป่า)

---

## ⭐ กฎมาตรฐาน (DEFAULT)

> **เวลาเรียน: normalize เป็น ๑ ชั่วโมง/แผน รวม ๑๐ ชั่วโมง/หน่วย**
>
> ตั้งค่าใน `scripts/build_docx.mjs`: `PLAN_HOURS = "๑"`, `UNIT_TOTAL_HOURS = "๑๐"`

## ๑. ข้อมูลภาพรวมเอกสาร

| รายการ | ค่า |
|--------|-----|
| ขนาดกระดาษ | A4 (210 × 297 mm) |
| จำนวนหน้า PDF ต้นฉบับ | ๒๖ หน้า |
| จำนวนแผนใน PDF ต้นฉบับ | **๑๓ แผน** (เวลาเดิมรวม ๑๙ ชม.) |
| **จำนวนแผนในไฟล์ docx** | **๑๐ แผน** (ยุบเหลือ ๑๐ ตามที่ผู้ใช้เลือก) |
| **เวลาต่อแผน** | **๑ ชั่วโมง/แผน** (normalize) |
| **เวลาเรียนรวมหน่วย** | **๑๐ ชั่วโมง** (normalize) |
| กลุ่มสาระ | ภาษาไทย |
| ชั้น | ประถมศึกษาปีที่ ๒ |
| ผู้สอน | นางสาวทักษพร การสนธิ์ |

### การยุบจาก ๑๓ → ๑๐ แผน

PDF ต้นฉบับมี ๑๓ แผน (รวมเวลาเดิม ๑๙ ชม.) — ผู้ใช้เลือก **"ยุบเหลือ ๑๐ แผน รวม ๑๐ ชม."**

แผน ๑–๑๐ (เวลาเดิมรวม ๒+๒+๒+๑+๑+๑+๑+๑+๑+๒ = ๑๔ ชม.) เป็น **แกนหลักของหน่วย**
— ขนานกับบทที่ ๓ พอดี (๑๐ แผน / ๑๔ ชม. เหมือนกัน) จึง **เก็บแผน ๑–๑๐** แล้ว normalize เป็น ๑ ชม./แผน

แผน ๑๑–๑๓ เป็น **แผนอ่านเสริม** จึง **ตัดออก**:
- ๑๑ การอ่านจับใจความ นิทานอ่านใหม่
- ๑๒ การอ่านนิทาน
- ๑๓ การอ่านเสริมบทเรียน

### รายการแผนใน docx (๑๐ แผน)

| แผน | เรื่อง | เวลา |
|---|---|---|
| ๑ | เรียนรู้คำ นำเรื่อง (ตอน ๑) | ๑ ชม. |
| ๒ | เรียนรู้คำ นำเรื่อง (ตอน ๒) | ๑ ชม. |
| ๓ | การอ่านออกเสียง | ๑ ชม. |
| ๔ | การอ่านจับใจความ | ๑ ชม. |
| ๕ | คำศัพท์จากบทเรียน | ๑ ชม. |
| ๖ | สระอึ ( –ึ ) และสระอือ ( –ือ ) | ๑ ชม. |
| ๗ | คำควบกล้ำ (คว กว ขว) | ๑ ชม. |
| ๘ | การแต่งประโยค | ๑ ชม. |
| ๙ | อ่านคล่อง ร้องเล่น “กลัวทำไม” | ๑ ชม. |
| ๑๐ | ชวนคิด ชวนทำ มาตราตัวสะกด (แม่กด/แม่เกอว) | ๑ ชม. |

**รวม:** ๑๐ ชั่วโมง

## ๒. ฟอนต์

- PDF ฝัง glyph เป็น PUA มาตรฐาน TH SarabunPSK → normalize เป็น Unicode ปกติ (เหลือ ๐ ตัว PUA)
- `extract_pdf.mjs` ตรวจหัวข้อ section แบบ **ไม่ผูกกับ font id** (ใช้ตำแหน่ง x < 85 + คำสำคัญหัวข้อ)
- **ฟอนต์เป้าหมายใน DOCX:** `TH SarabunPSK` ขนาด **16 pt**

---

## ๓. ระยะขอบ + โครงสร้าง

- ระยะขอบ ~2.5 cm ทุกด้าน (1417 twips)
- โครงสร้างแต่ละแผน: Header ๓ บรรทัด (2-column) + เส้นจุดประ + สาระ/มาตรฐาน + หัวข้อ ๑–๘ (จาก PDF)
  + หัวข้อ ๙ ตารางวัดผล + หัวข้อ ๑๐ + บันทึกหลังสอน + ลายเซ็น ๓ จุด (generic จาก ref_data.json)
- ๑ แผน = ๑ ชุด (Page Break ระหว่างแผน)

## ๔. ขั้นตอนการสร้าง (Pipeline)

ใช้ node_modules ที่ `บทที่ ๑ น้ำใส/แบบฝึกหัด/node_modules/` (docx, pdfjs-dist, jszip, pdf-to-img)

```
cd "บทที่ ๔ กลัวทำไม"
node scripts/extract_pdf.mjs    # PDF → pdf_data.json  (Plans found: 13, แผนละ 8 sections)
node scripts/build_docx.mjs     # → หน่วยการเรียนรู้ที่๔_กลัวทำไม.docx (เก็บแผน ๑–๑๐)
node scripts/validate_docx.mjs  # ตรวจ parts + 0 PUA + จำนวนแผน + เวลา
```

ผลลัพธ์: `หน่วยการเรียนรู้ที่๔_กลัวทำไม.docx` (๑๐ แผน, ๑๐ ชั่วโมง)

## ๕. แบบฝึกหัด (ใบงาน ๑๐ ใบ)

เนื้อหาใบงานดึงจาก **๖. กิจกรรมการเรียนรู้** ของแต่ละแผน + เนื้อหาจริงในหนังสือเรียน
(รูปหน้าหนังสือ `กลัวทำไม/1.png … 9.png`) เช่น คำศัพท์ นำเรื่อง, เรื่อง “กลัวทำไม” (ถ้ำ หินงอก
หินย้อย พนมมืออธิษฐาน ตะไคร้หอมไล่ยุง), กฎสระอือ (มี/ไม่มี อ เคียง), เพลงคำควบกล้ำ,
มาตราแม่กด/แม่เกอว, บทอ่านคล่องร้องเล่น “กลัวทำไม”, กิจกรรมคิดเปรียบเทียบ

```
cd "บทที่ ๔ กลัวทำไม/แบบฝึกหัด"
node generate_worksheets.mjs    # → worksheet_X.html + ใบงานที่_X_บทที่_๔_กลัวทำไม.pdf (X = ๑..๑๐)
```

- สไตล์เดียวกับบทที่ ๒/๓ (การ์ดสี + แบนเนอร์ + footer คะแนน/ลายเซ็น)
- แปลง HTML → PDF ด้วย Microsoft Edge headless
- มี PDF ใบงานแยก ๑๐ ไฟล์ในโฟลเดอร์ และ **ฝังลงในไฟล์ docx ท้ายแต่ละแผนด้วย** (เหมือนบทที่ ๒)

### ฝังใบงานเข้า docx

```
cd "บทที่ ๔ กลัวทำไม"
node scripts/build_docx.mjs        # สร้าง docx ก่อน
node scripts/embed_worksheets.mjs  # แล้วฝังรูปใบงาน (render PDF→PNG เต็มหน้า แทรกท้ายแผนที่ตรงกัน)
```

> ต้องรัน `build_docx.mjs` ก่อน `embed_worksheets.mjs` เสมอ (embed แก้ไฟล์ docx ในที่ — รัน build ทีหลังจะทับ)
> ใบงาน ๑๐ ใบ = ๑๙ หน้ารูป (ใบ ๓ = ๑ หน้า, ที่เหลือ ๒ หน้า) → docx ~2.8 MB

## ๕.๑ การจัดหน้า (keep-together) — DEFAULT

แก้ปัญหาหัวข้อค้างท้ายหน้า / ตารางถูกตัดข้ามหน้า ใน `build_docx.mjs`:
- **ตาราง ๙ การวัดผล:** ทุก `TableRow` ใส่ `cantSplit: true` + ทุก paragraph ใน cell ใส่ `keepNext: true`
  → หัวข้อ ๙ + ตารางเป็นบล็อกเดียว ถ้าท้ายหน้าไม่พอ Word ดันทั้งบล็อกลงหน้าใหม่
- **บันทึกหลังแผนการจัดการเรียนรู้:** หัวข้อใส่ `pageBreakBefore: true` → ขึ้นหน้าใหม่ รวมข้อ ๑/๒/๓ + ลายเซ็นไว้ด้วยกัน
- หัวข้อ section ๑–๘/๑๐ และ signature tables มี `keepNext`/`cantSplit` อยู่แล้ว
- ตรวจ XML: `cantSplit` = ๗๐ (๗ แถว/แผน ×๑๐), `pageBreakBefore` = ๑๐ (๑/แผน)
