# Layout / รูปแบบเอกสาร — หน่วยการเรียนรู้ที่ ๓ เรื่อง ครัวป่า

> ไฟล์นี้บันทึกรูปแบบของไฟล์ PDF ต้นฉบับ `หน่วยการเรียนรู้ที่ ๓ ครัวป่า (1).pdf`
> และขั้นตอนแปลงเป็น DOCX (รูปแบบเดียวกับบทที่ ๑ น้ำใส / บทที่ ๒ ใจหาย)

---

## ⭐ กฎมาตรฐาน (DEFAULT) สำหรับการแปลงต่อจากนี้

> **เวลาเรียน: ทุกหน่วยให้ normalize เป็น ๑ ชั่วโมง/แผน รวม ๑๐ ชั่วโมง/หน่วย**
>
> - เก็บแผนให้ครบตามต้นฉบับ (ไม่ตัด ยกเว้นมีแผนซ้ำกันจริง)
> - แต่ละแผนตั้งเวลาเป็น **เวลา ๑ ชั่วโมง** เสมอ
> - หัวหน่วย (Unit header) ตั้งเป็น **เวลา ๑๐ ชั่วโมง** เสมอ
> - ค่าเวลาเดิมใน PDF (เช่น บทที่ ๓ มี ๒/๒/๒/๑/… รวม ๑๔ ชม.) **ไม่ต้องใช้** — override ด้วยค่ามาตรฐาน
>
> ตั้งค่าใน `scripts/build_docx.mjs`: `PLAN_HOURS = "๑"`, `UNIT_TOTAL_HOURS = "๑๐"`

## ๒. ฟอนต์

- PDF ฝัง glyph เป็น Private Use Area (PUA) มาตรฐาน TH SarabunPSK → ต้อง normalize เป็น Unicode ปกติ
- **ฟอนต์หัวข้อใน PDF บทนี้คือ `g_d0_f1`** (ต่างจากบทที่ ๒ ที่ใช้ `g_d0_f2`)
  → `extract_pdf.mjs` จึงตรวจหัวข้อ section แบบ **ไม่ผูกกับ font id** (ใช้ตำแหน่ง x < 85 + คำสำคัญหัวข้อแทน)
- **ฟอนต์เป้าหมายใน DOCX:** `TH SarabunPSK` ขนาด **16 pt**

## ๓. ระยะขอบ + โครงสร้าง

- ระยะขอบ ~2.5 cm ทุกด้าน (1417 twips)
- โครงสร้างแต่ละแผน: Header ๓ บรรทัด (2-column) + เส้นจุดประ + สาระ/มาตรฐาน + หัวข้อ ๑–๘ (จาก PDF)
  + หัวข้อ ๙ ตารางวัดผล (จาก reference) + หัวข้อ ๑๐ + บันทึกหลังสอน + ลายเซ็น ๓ จุด (จาก reference)
- ๑ แผน = ๑ ชุด (Page Break ระหว่างแผน)
- หัวข้อ ๙ และ ๑๐ + ส่วนท้าย: ใช้เนื้อหา generic ที่ hardcode ใน `build_docx.mjs` (เหมือนบทที่ ๑/๒)

## ๔. ขั้นตอนการสร้าง (Pipeline)

ใช้ node_modules ที่อยู่ที่ `บทที่ ๑ น้ำใส/แบบฝึกหัด/node_modules/` (docx, pdfjs-dist)

```
cd "บทที่ ๓ ครัวป่า"
node scripts/extract_pdf.mjs    # PDF → pdf_data.json (+ pdf_raw.json) — ควรได้ Plans found: 10, แผนละ 8 sections
node scripts/build_docx.mjs     # pdf_data.json + ref_data.json → หน่วยการเรียนรู้ที่๓_ครัวป่า.docx
```

ผลลัพธ์: `หน่วยการเรียนรู้ที่๓_ครัวป่า.docx` (๑๐ แผน, ๑๐ ชั่วโมง)

> ขอบเขตบทนี้ = **แผนการสอนอย่างเดียว** (ไม่สร้าง/ไม่แทรกใบงาน ต่างจากบทที่ ๒ ที่มีใบงานแทรกท้ายแผน)

## ๕. ไฟล์ในโฟลเดอร์ scripts/

| ไฟล์ | หน้าที่ |
|------|--------|
| `extract_pdf.mjs` | สกัดข้อความจาก PDF + normalize PUA → `pdf_data.json` |
| `build_docx.mjs` | สร้าง DOCX จาก `pdf_data.json` + `ref_data.json` |
| `ref_data.json` | ข้อมูลผู้สอน/ลายเซ็น (copy จากบทที่ ๒) |
| `pdf_data.json` / `pdf_raw.json` | ผลสกัด (generated) |
