# โปรเจ็คแผนการจัดการเรียนรู้ภาษาไทย ป.๒

> ชุดเอกสารแผนการจัดการเรียนรู้กลุ่มสาระภาษาไทย ระดับชั้นประถมศึกษาปีที่ ๒ จำนวน ๑๐ บท
> แต่ละบทประกอบด้วย ๑๐ แผนการสอนย่อย
> ใช้บทที่ ๑ น้ำใส เป็น *reference template* สำหรับสร้างบทที่ ๒–๑๐

## ๑. ภาพรวมโปรเจ็ค

| รายการ | ค่า |
|--------|-----|
| กลุ่มสาระ | ภาษาไทย |

## ๓. Workflow การสร้างหนึ่งบทใหม่ (๗ ขั้นตอน)

เมื่อต้องการเพิ่มบทใหม่ (เช่น บทที่ ๒) ให้ทำตามลำดับนี้:

### ขั้นที่ ๑ — ผู้ใช้วาง PDF
ผู้ใช้นำไฟล์ PDF ต้นฉบับของบทใหม่ไปวางที่:
```
d:\แผน\บทที่ <เลขไทย>\<ชื่อบท>.pdf
```
ตัวอย่าง: `d:\แผน\บทที่ ๒\หน่วยการเรียนรู้ที่ ๒ <ชื่อ>.pdf`

### ขั้นที่ ๒ — Claude อ่าน PDF แล้วสร้าง `Layout.md`
อ่านรูปแบบ (format) ของ PDF อย่างละเอียด:
- ฟอนต์ที่ใช้ (TH SarabunPSK, Cordia New ฯลฯ)
- ขนาดฟอนต์ของแต่ละองค์ประกอบ
- ระยะขอบ (margin) บน/ล่าง/ซ้าย/ขวา
- การจัด layout ของ header, footer, ตาราง
- การย่อหน้า (indentation) แต่ละระดับ

บันทึกเป็นไฟล์: `d:\แผน\บทที่ <X>\Layout.md`

### ขั้นที่ ๓ — Claude อ่าน PDF แล้วสร้าง `Content.md`
อ่านเนื้อหาทั้ง ๑๐ แผนการสอน — โดย PDF จะมีเฉพาะ **หัวข้อที่ ๑ ถึง ๘** เท่านั้น:
1. สาระสำคัญ
2. ตัวชี้วัด
3. จุดประสงค์การเรียนรู้
4. คุณลักษณะอันพึงประสงค์
5. สาระการเรียนรู้
6. กิจกรรมการเรียนรู้
7. สื่อ / แหล่งการเรียนรู้
8. วัดผลประเมินผล

บันทึกเป็นไฟล์: `d:\แผน\บทที่ <X>\Content.md`

### ขั้นที่ ๔ — Claude สร้างไฟล์ docx
ใช้ `Layout.md` + `Content.md` รวมกันสร้าง:
```
d:\แผน\บทที่ <X>\หน่วยการเรียนรู้ที่<X>_<ชื่อ>_final5.docx
```

### ขั้นที่ ๕ — เพิ่มหัวข้อที่ ๙ และ ๑๐
หัวข้อที่ ๙ และ ๑๐ ของทุกบทจะเหมือนกัน — copy โครงสร้างและเนื้อหาจาก:
```
d:\แผน\บทที่ ๑ น้ำใส\หน่วยการเรียนรู้ที่๑_น้ำใส_final5.docx
```
**ผู้ใช้ไม่ต้องให้ข้อมูลหัวข้อ ๙ และ ๑๐ — Claude ดึงจาก reference เอง**

### ขั้นที่ ๖ — เปลี่ยนชื่อ ๓ ตำแหน่งให้ตรงกับ reference
ดึงชื่อ ๓ ตำแหน่งจาก `final5.docx` ของบทที่ ๑ มาใส่ในบทใหม่:
- ชื่อ**ผู้ฝึกสอน** = นางสาวทักษพร การสนธิ์ (ครูผู้สอน + ผู้บันทึก)
- ชื่อ**รองผู้อำนวยการ** = () ว่าง (รองผู้อำนวยการโรงเรียนบ้านหนองเต่า)
- ชื่อ**ครูประจำชั้น** = () ว่าง (หัวหน้ากลุ่มสาระการเรียนรู้)

**ผู้ใช้ไม่ต้องพิมพ์ชื่อทั้ง ๓ — Claude อ่านจาก final5.docx โดยอัตโนมัติ**

#### ๖.๑ ฟอร์มลายเซ็น ๓ บล็อก (ต้องจัดกลางหน้า + ชื่อ/ตำแหน่งกลางใต้เส้น)

ทุกแผนมีบล็อกลายเซ็น ๓ ชุด ต้องสร้างด้วย **ตารางไม่มีขอบ + alignment CENTER** ภายในเซลล์:

| บล็อก | จำนวนเซลล์ | เนื้อหาแต่ละบรรทัด (จัดกลาง) |
|------|-----------|--------------------------------|
| บน — หลังหัวข้อ ๑๐ | ๑ เซลล์ × ๑๐๐% | `ลงชื่อ........ครูผู้สอน` / `(ชื่อ)` |
| กลาง — หลังบันทึกหลังสอน | ๑ เซลล์ × ๑๐๐% | `ลงชื่อ........ผู้บันทึก` / `(ชื่อ)` / `ครูผู้สอน` |
| ล่าง — ๒ ลายเซ็น | ๒ เซลล์ × ๕๐%/๕๐% | (ซ้าย) `ลงชื่อ.......` / `()` / `หัวหน้ากลุ่มสาระการเรียนรู้` <br> (ขวา) `ลงชื่อ.......` / `()` / `รองผู้อำนวยการโรงเรียนบ้านหนองเต่า` |

**คุณสมบัติบังคับของตารางลายเซ็น:**
- `borders: NONE` (ไม่มีเส้นขอบให้เห็น)
- `TableRow.cantSplit: true` (ห้ามตัดบล็อกข้ามหน้า)
- `AlignmentType.CENTER` บนทุกย่อหน้าในเซลล์
- `keepNext: true` บนทุกบรรทัดยกเว้นบรรทัดสุดท้ายของเซลล์

#### ๖.๒ การจัดหน้าไม่ให้หัวข้อ/ตารางขาดหน้า (keep-together) — บังคับทุกบท

ป้องกันปัญหา **หัวข้อค้างท้ายหน้าแล้วเนื้อหาขึ้นหน้าใหม่ / ตารางถูกตัดครึ่ง**
หลักการ: ถ้าบล็อกหนึ่งจะถูกตัด ให้ **เลื่อนหัวข้อลงไปอยู่หน้าเดียวกับเนื้อหา** (พฤติกรรม `keepNext` ของ Word)
ตั้งค่าใน `build_docx.mjs`:

| องค์ประกอบ | property | ผล |
|-----------|----------|-----|
| ทุก section (๑–๘) — `buildSection1` / `buildSection4` / `buildGenericSection` | `keepNext: true` ทุกบรรทัด **ยกเว้นบรรทัดสุดท้ายของ section** + `keepLines: true` | หัวข้อ + ทุกข้อย่อยเป็นบล็อกเดียว ถ้าท้ายหน้าไม่พอ Word ดันทั้ง section ลงหน้าใหม่ (แต่ละ section < ๑ หน้า จึงพอดีเสมอ) บรรทัดสุดท้ายไม่ผูก → ยอมให้ขึ้นหน้าระหว่าง section ได้ |
| ตาราง ๙ การวัดผล — `buildSection9` | `TableRow.cantSplit: true` ทุกแถว + `keepNext: true` ทุกย่อหน้าในเซลล์ (ผ่าน helper `tp()`) | หัวข้อ ๙ + ทั้งตารางผูกเป็นบล็อกเดียว ไม่ถูกตัดข้ามหน้า |
| บันทึกหลังแผนการจัดการเรียนรู้ — `buildTrailing` | หัวข้อใส่ `pageBreakBefore: true` | ขึ้นหน้าใหม่เสมอ รวมข้อ ๑ ผลการสอน / ๒ ปัญหาและอุปสรรค / ๓ ข้อเสนอแนะ + ลายเซ็นไว้ด้วยกัน |
| ตารางลายเซ็น ๓ บล็อก | `TableRow.cantSplit: true` (มีอยู่แล้ว) | บล็อกลายเซ็นไม่ขาดหน้า |

**สำคัญ:** บรรทัด **สุดท้าย** ของแต่ละ section ต้อง `keepNext: false` เสมอ — ไม่งั้นจะผูกติดกับหัวข้อ section ถัดไปเป็นโซ่ยาวทั้งแผน

**ตรวจสอบ (headless):** เปิด docx ด้วย jszip อ่าน `word/document.xml` แล้วนับ
`<w:cantSplit>` = ๗ แถว/แผน × จำนวนแผน (๔ แถวตาราง ๙ + ๓ แถวลายเซ็น) , `<w:pageBreakBefore>` = ๑/แผน
> หมายเหตุ: เครื่องสร้างไม่มี Word/LibreOffice จึงตรวจได้แค่ว่า property ถูกเขียนลง XML — ผลการแบ่งหน้าจริงต้องเปิดใน Word ดู

### ขั้นที่ ๗ — สร้างใบงาน (worksheets)
สร้างใบงาน ๑๐ ใบสำหรับบทใหม่ ลงในโฟลเดอร์ย่อย:
```
d:\แผน\บทที่ <X>\แบบฝึกหัด\worksheet_<เลขไทย>.html
                          \ใบงานที่_<เลขไทย>_บทที่_<X>_<ชื่อ>.pdf
```
อ้างอิงรูปแบบจาก `บทที่ ๑ น้ำใส\แบบฝึกหัด\`

**ขั้นตอนการสร้างใบงาน:**
1. เขียนสคริปต์ `generate_worksheets.mjs` กำหนดเนื้อหาทั้ง ๑๐ ใบ (ใช้ template HTML + CSS เดียวกับบทที่ ๑)
2. รัน script → สร้าง HTML ๑๐ ไฟล์
3. ใช้ Microsoft Edge headless แปลง HTML → PDF อัตโนมัติ:
   ```
   msedge --headless=new --no-margins --print-to-pdf="output.pdf" --print-to-pdf-no-header "file:///path.html"
   ```

#### ๗.๑ ตารางแจกลูกสะกดคำ — ต้องมีเส้น ๓ ระดับแบบกระดาษคัดลายมือ

CSS class `.spell td.line` ต้องใช้รูปแบบ **เส้น ๓ ระดับ** (บน + กลางอ่อน + ล่าง) เพื่อให้เด็ก ป.๒ เขียนตามแนว:

```css
.spell td.line{
  background:
    linear-gradient(#22a06b,#22a06b) 0 0/100% 1.2px no-repeat,  /* เส้นบน (baseline-top) */
    linear-gradient(#a3d9bb,#a3d9bb) 0 50%/100% 1px no-repeat,  /* เส้นกลาง (guide line - อ่อนกว่า) */
    linear-gradient(#22a06b,#22a06b) 0 100%/100% 1.2px no-repeat;/* เส้นล่าง (baseline-bottom) */
  height:38px;
}
```

ใช้ได้กับใบงานที่มีกิจกรรม "แจกลูกสะกดคำ", "เติมตัวสะกด", "อ่านสะกดคำ", "เขียนคำที่อ่านได้", หรือกิจกรรมใดๆ ที่นักเรียนต้องเขียนคำตอบเป็นคำ

#### ๗.๒ กฎเนื้อหาใบงาน (Worksheet content rules)

ใบงานต้องเป็น **แบบฝึกที่นักเรียนทำได้ด้วยตนเอง** ไม่ใช่การคัดลอกหนังสือ:

1. **ห้ามใส่กิจกรรม "ดูภาพ/อ่านตามหนังสือ"** — กิจกรรมที่แค่เอาภาพหน้าหนังสือมาแปะแล้วบอกให้ "อ่านคำใต้ภาพ" ให้ตัดออก เพราะนักเรียนเปิดอ่านในหนังสือเรียนได้เอง ใบงานควรเป็นกิจกรรมที่ต้องคิด/เขียน/ทำ (เติมคำ โยงเส้น แยกประเภท แต่งประโยค ตอบคำถาม คัดคำ ฯลฯ)
2. **ห้ามมีคำถามซ้ำกัน** — ถ้ามีคำถาม/กิจกรรมที่ถามสิ่งเดียวกัน ไม่ว่าจะ **ในใบเดียวกัน** หรือ **ข้ามใบงาน** ให้เลือกไว้เพียงที่เดียว อย่าให้มีสองอัน
   - ตัวอย่าง: บทที่ ๓ ใบงานที่ ๓ — "อาหารที่ทำกันในครัวป่ามีอะไรบ้าง" ซ้ำกับการ์ดแผนภาพความคิด "เขียนชื่ออาหารที่ปรุงในเรื่องครัวป่า" → เก็บไว้เฉพาะแผนภาพความคิด
   - ตัวอย่าง: การ์ด "ข้อคิด + นำไปใช้" ซ้ำกันได้ง่ายเมื่อหลายใบเป็นเรื่องเดียวกัน → เก็บไว้ในใบที่จุดประสงค์ระบุ "นำข้อคิดไปใช้" เท่านั้น
3. **รูปอ้างอิง** — ถ้าต้องใช้รูป ให้ดึงจากโฟลเดอร์รูปหนังสือเรียนของบทนั้น (เช่น `บทที่ ๓ ครัวป่า\ครัวป่า\`)

### ขั้นที่ ๘ — ฝัง PDF ใบงานเข้าใน docx
หลังสร้างใบงานครบ ใช้สคริปต์ `embed_worksheets.mjs`:
1. Render ทุก PDF เป็น PNG (page-by-page) ด้วย `pdf-to-img`
2. ฝัง PNG เป็น floating image แบบเต็มหน้า A4 (`positionH/V = page 0,0`, `cx/cy = A4_W/A4_H`)
3. แทรกท้ายแต่ละแผน (หลัง page break) → ใบงานหน้า ๑ → page break → ใบงานหน้า ๒ → page break → แผนถัดไป

**ผลลัพธ์:** docx ขนาด ~3.4 MB ต่อบท (รวมรูปใบงาน ๒๐ รูป)

### ๔.๑ มาตรฐานบังคับ (Hard Requirements)

ทุกบทที่สร้างใหม่ **ต้องเป็นไปตาม** มาตรฐานต่อไปนี้:

1. **ฟอร์มลายเซ็น ๓ บล็อก** — จัดกลางหน้า + ชื่อ/ตำแหน่งจัดกลางใต้เส้น (ใช้ตารางไม่มีขอบ) → ดูข้อ ๓ ขั้นที่ ๖.๑
2. **ตารางแจกลูกสะกดคำ** — มีเส้น ๓ ระดับ (บน + กลางอ่อน + ล่าง) ความสูง 38px → ดูข้อ ๓ ขั้นที่ ๗.๑
3. **หัวข้อกับเนื้อหา/ตาราง** — ห้ามตัดข้ามหน้า: ทุก section เป็นบล็อก `keepNext`, ตาราง ๙ ใส่ `cantSplit` ทุกแถว, บันทึกหลังแผนใส่ `pageBreakBefore` → ดูข้อ ๓ ขั้นที่ ๖.๒
4. **เส้นจุดประที่เขียนได้** — ต้องเต็มความกว้างหน้ากระดาษ (ใช้ tab leader `LeaderType.DOT` ไม่ใช่ fixed-length dots)
5. **หัวข้อ ๙ และ ๑๐** — copy จากบทที่ ๑ เสมอ (ไม่ต้องเขียนใหม่)
6. **ใบงานในรายงาน** — ฝัง PDF เป็น floating image เต็มหน้า A4 ท้ายแต่ละแผน
7. **ใบงานต้องเป็นแบบฝึก ไม่ใช่อ่านตามหนังสือ** — ห้ามมีกิจกรรม "ดูภาพในหนังสือแล้วอ่านคำ" → ดูข้อ ๓ ขั้นที่ ๗.๒
8. **ห้ามคำถามซ้ำ** — คำถาม/กิจกรรมที่ถามสิ่งเดียวกัน (ในใบเดียวกันหรือข้ามใบ) ให้เก็บไว้เพียงที่เดียว → ดูข้อ ๓ ขั้นที่ ๗.๒
